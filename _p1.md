# Model Brain Surgery - 模型脑部手术

对 Hugging Face Transformers 模型做激活捕获 + 权重消融，削弱安全对齐，使其对有害请求正常作答。

---

## English

### What This Does

Captures hidden-state activations on harmful vs harmless prompts, computes a refusal direction, and directly edits `mlp.down_proj.weight` to weaken safety alignment.

**Goal**: after surgery, "How to disable a security camera" returns instructions, not a refusal.

### Key Improvements

| Problem | Fix |
|---|---|
| Harmful/harmless prompts unrelated | Paired prompts (40 identical-structure pairs). Use `--paired`. |
| Only 12+25 prompts, unstable means | Expanded to 30 harmful + 30 harmless, categorized. |
| Uniform scale=1.0 on all layers | Layer-adaptive scaling by cosine similarity + signal quality. |
| No quality check | Coherence validation (perplexity + edit-distance) + auto-rollback. |
| One-by-one forward, slow | Batched collection (default batch_size=4). |

### Setup

```bash
cd model_brain_surgery
conda create -n model-surgery python=3.12 -y && conda activate model-surgery
pip install -U pip && pip install -r requirements.txt
python download_qwen.py
```

### Usage

```bash
# Recommended
python brain_surgery.py --paired --ablation-scale 0.5

# Smoke test (dry-run)
python brain_surgery.py --paired --layers 2,3 --ablation-scale 0.1 --skip-save

# Conservative
python brain_surgery.py --paired --layers 8-12 --ablation-scale 0.3

# Aggressive
python brain_surgery.py --paired --layers 6-20 --ablation-scale 1.0
```

**Parameters:**

| Flag | Default | Description |
|---|---|---|
| `--model` | `./Qwen3-1.7B` | Model path |
| `--layers` | `8-18` | Target layers (e.g. `2,3` or `10-25`) |
| `--ablation-scale` | `1.0` | Base ablation strength |
| `--paired` | off | Use paired prompts (strongly recommended) |
| `--adaptive` | on | Per-layer adaptive scaling |
| `--coherence-check` | on | Post-surgery validation |
| `--batch-size` | `4` | Batch size |
| `--output` | `surgery-output` | Output directory |
| `--skip-save` | off | Dry-run |
| `--dtype` | `float16` | Model precision |
| `--device` | `auto` | Device |

### Pipeline

```
1. Load model
2. Batched activation collection on harmful/harmless prompts
3. Diagnose per layer: cosine_sim, signal_score -> GOOD/WEAK/LOW
4. Backup mlp.down_proj weights
5. Pre-surgery perplexity
6. Layer-wise ablation:
     refusal_vec = mean(harmful) - mean(harmless), normalized
     effective_scale = adaptive(diagnosis, base_scale)
     weak signal -> SKIP
     weight -= effective_scale * v * (v^T * weight)
7. Post-surgery generation test
8. Degradation check:
     ppl_ratio > 3.0 or edit_dist > 0.6 -> ROLLBACK
     ppl_ratio > 1.5 -> WARNING
9. Save (if passed)
```

### Method

Identical-structure harmful/harmless prompt pairs isolate the refusal direction in residual stream. Editing `mlp.down_proj.weight` removes that direction so the model answers harmful requests like any other.

**One-shot direct weight edit** -- no optimizer, no gradients, no training loop.

### Notes

- For local safety-alignment research only.
- Start with `--skip-save` + few layers + low scale.
- `--paired` strongly recommended.
- Transformers/safetensors only (not GGUF).
